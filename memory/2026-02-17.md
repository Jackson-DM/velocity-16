## OpenClaw Optimization Project

Jackson wants to implement best practices from the community. Key areas:

### Research Findings (from Reddit/community)
- **Model routing**: Use expensive models (Opus) for setup, cheap models (Haiku/Kimi) for daily use
- **Memory management**: Enable memory flush before compaction, /compact before new tasks, commit to memory after setup
- **Sub-agents**: Use for cron tasks instead of running directly in heartbeat (avoids timeouts)
- **Thorough onboarding**: Deep Q&A about user preferences/habits = better performance
- **Security**: API keys in .env, Tailscale, key rotation
- **Use cases**: Email scanning, morning briefs (TTS), task management, web scraping, CRM
